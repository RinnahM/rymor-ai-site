RAG architecture — retrieval, reasoning & memory

The Problem

Apex Home Repair's tribal knowledge — past project specs, repair manuals, pricing models — was scattered across emails and PDFs. Employees spent hours searching for answers instead of doing repairs, creating high cognitive load and inconsistent information sharing across the team.

What I Built

A Retrieval-Augmented Generation (RAG) pipeline using Google Gemini as the reasoning engine to process complex, unstructured service documentation, paired with Qdrant as the vector database for sub-millisecond retrieval of specific procedures or past project data. A Postgres database gives the agent memory of prior conversations, so it holds context across a session instead of answering each question cold. The result acts as an expert teammate: it doesn't guess — it reads Apex's own files and answers technical questions, cross-references historical projects, and pulls exact specs in seconds.
